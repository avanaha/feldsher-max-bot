# Security Documentation

## Обзор системы безопасности

Данный документ описывает меры безопасности, реализованные в боте «Фельдшеръ.Рядом» версии 11.0.

---

## 1. Защита персональных данных

### 1.1 Шифрование

**Реализовано шифрование номеров телефонов в базе данных:**

- Алгоритм: AES-256-CBC
- Ключ шифрования: генерируется из `ENCRYPTION_KEY` через scrypt
- Каждый номер шифруется с уникальным IV (Initialization Vector)
- Формат хранения: `IV:encrypted_data` (hex)

```typescript
// Пример шифрования
const encrypted = encrypt('+7 (999) 123-45-67');
// Результат: 'a1b2c3d4...:e5f6g7h8...'

// Пример расшифровки
const phone = decrypt(encrypted);
// Результат: '+7 (999) 123-45-67'
```

### 1.2 Валидация входных данных

| Поле | Валидация |
|------|-----------|
| Имя | До 100 символов, санитизация HTML |
| Телефон | 10-15 цифр, форматирование в единый формат |
| Стаж | Число от 0 до 50 |
| Район | Только из предустановленного списка |
| Резюме URL | Проверка домена, поддержка текстового описания |
| Вопрос | До 1000 символов |

### 1.3 Санитизация

Все пользовательские данные проходят санитизацию:
- Удаление HTML/JS тегов
- Обрезка до максимальной длины
- Удаление потенциально опасных паттернов (`<script`, `javascript:`, `on\w+=`)

## 2. Rate Limiting

### 2.1 Параметры

| Параметр | Значение |
|----------|----------|
| Окно | 60 секунд |
| Максимум запросов | 30 в минуту |
| Предупреждение | После 20 запросов |

### 2.2 Исключения

**Пользователи в активной анкете освобождены от rate limiting:**

```typescript
// Пользователи в процессе заполнения анкеты
const activeQuestionnaires = new Set<number>();
```

Это позволяет беспрепятственно проходить все шаги анкеты.

### 2.3 Поведение

- Не блокирует пользователя
- Молча игнорирует лишние запросы
- Отправляет уведомление админу при превышении лимита

## 3. Логирование и мониторинг

### 3.1 Ротация логов

| Параметр | Значение |
|----------|----------|
| Срок хранения | 7 дней |
| Частота проверки | Раз в сутки |
| Расположение | `/app/data/logs/` |

### 3.2 Типы логов

**Основной лог (`bot.log`):**
- INFO: Стандартные операции
- WARN: Предупреждения
- ERROR: Ошибки
- DEBUG: Отладочная информация

**Security лог (`security.log`):**
- CONSENT_GRANTED/DENIED
- WAITLIST_START/SUBMITTED
- FELDSHER_START/SUBMITTED
- QUESTION_START/SUBMITTED
- SUSPICIOUS_ACTIVITY
- DATA_REVOKED

Автоматические уведомления отправляются при:

```
GET /health
```

### 6.2 Response

```json
{
  "status": "ok",
  "timestamp": "2024-01-01T00:00:00.000Z",
  "version": "11.0"
```

## 7. Обработка блокировок

### 7.1 chat.denied

При блокировке бота пользователем:
1. Ловится ошибка `chat.denied`
2. Удаляются все данные пользователя из БД
3. Записывается лог

### 7.2 Отзыв согласия

```
/revoke — удалить свои данные
```

## 9. Рекомендации по развёртыванию

1. **Установите уникальный ENCRYPTION_KEY** в продакшене
2. **Используйте HTTPS** для всех соединений
3. **Ограничьте доступ** к `/app/data/` на уровне файловой системы
4. **Настройте мониторинг** логов безопасности
5. **Регулярно проверяйте** бэкапы
